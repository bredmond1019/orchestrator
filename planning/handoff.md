## Context the next agent needs

- **`Workflow.run()` parses the inbound dict** into the workflow's `event_schema` before
  handing it to nodes (`task_context.event = self.workflow_schema.event_schema(**event)`).
  This means nodes always receive a Pydantic model object in production, never a raw dict.
  The `isinstance(event, dict)` branch in `_build_initial_messages` was defensive code
  from unit-test context; `TestBuildInitialMessagesWithPydanticEvent` now covers the real
  production path explicitly.
- **`schema_registry.py` step is mandatory** for every new workflow. The api-reference
  documents this as step 3 of "Adding a New Entry" (WorkflowRegistry section). The SDLC
  pipeline did not check for it — it only verified the WorkflowRegistry enum entry, not the
  schema dispatcher map. Consider adding a harness.json check for this in the future.
- **`ToolUseNode.process()` does not catch exceptions from `handle_tool_call`**. If a tool
  raises (e.g. Tavily network error), it propagates and kills the workflow. The new
  `TestSearchServiceExceptionPropagation` test documents this as the known behavior.
- **Local run recipe**: Postgres + Redis running locally; real API keys (`VOYAGE_API_KEY`,
  `ANTHROPIC_API_KEY`) are in root `.env.local`, not `app/.env`. Launch with
  `set -a; source ../.env.local; set +a` from `app/` before `uv run`. `TAVILY_API_KEY`
  needed for live `SearchService` calls. `TOOL_USE_MODEL` env var controls the model for
  `CompanyResearchNode` (defaults to `claude-haiku-4-5-20251001`).
- Recurring `VIRTUAL_ENV` warning is from a stale `agentic-portfolio/orchestration/.venv`
  path; harmless — `unset VIRTUAL_ENV` before `uv run` to silence it.
